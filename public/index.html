Provision your SplitVault (and payment asset)

If a buyer’s first payment failed because your SplitVault or vault token account is not on-chain yet, use this flow. You only need a Solana browser wallet (Phantom, Solflare, Backpack, Trust, or any compatible adapter) and a little SOL on the same network as this facilitator (see the banner / health strip below).

Steps: pick an asset (default USDC for typical agent flows; SOL, …) → 🔗 Connect wallet → Provision. Re-running for the same wallet + asset is a no-op when already on-chain. Presets like USDC are cluster-aware: this facilitator resolves them to the correct SPL mint for its Solana cluster (e.g. Circle devnet vs mainnet USDC are different addresses—the server picks the right one; you do not paste a mint for normal USDC).

Optional: paste any SPL mint to override the list (required for USDT on devnet, custom test mints, memecoins, etc.).
