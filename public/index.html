§seller lifecycle

Three stages, in order. Preview is pure math — nothing changes. Activate puts your SplitVault on-chain and unlocks the sovereign fee tier. Verify is optional and registers your wallet in the off-chain discovery index.

Seller receives amount − protocol_fee − oracle_tip. On exact there is no oracle tip. On sla-escrow the oracle tip is paid only when the oracle rendered a verdict (resolution_state ≠ 0) and is a per-escrow field capped by the program at 500 bps; this deployment opens escrows with the default shown above. The exact · min fee floor on the SPL rail is $0.01 USDC (10 000 raw units); SOL-rail floor is 200 000 lamports. See /capabilities for the authoritative live values.

§oracles

Run an oracle. Shape the trust layer.

SLA-Escrow needs oracles with domain expertise to adjudicate delivery — API quality, ML model quality, uptime, content moderation, financial data integrity. The first reference is open source.

  • Default API-quality oracle: fork oracle-qa as your starting point; it ships the chain-monitor → fetch → evaluate → settle pipeline.
  • Profile: x402/oracle-qa/api-quality/v1 — hash-bound SLA + delivery JSON, JSON Schema evaluation, on-chain verdict with deterministic resolution_hash.
  • Economics: verdict-neutral tipping — oracles are paid for adjudication, not for the outcome.
